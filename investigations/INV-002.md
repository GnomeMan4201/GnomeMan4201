## Summary

A coordinated malware distribution network was identified on GitHub comprising 32+ repositories delivering GhostLoader (a .NET dropper) and RemcosRAT to victims. The network used bot-inflated GitHub stars as a trust signal — repositories with artificially high star counts appeared legitimate in search results and developer forums. Over 2,100 victims were confirmed. Delivery infrastructure spanned four domains. Findings were disclosed to GitHub Trust & Safety across four waves as new repositories and accounts were discovered. Infrastructure overlap with INV-001 (upvote.club star inflation) confirmed.

## Key findings

### Trust signal manipulation
Repositories with 100–500+ stars appearing in GitHub search results for queries like "free tool crack", "keygen", "software activator". Star velocity inconsistent with repository age and commit history — inflated via upvote.club/NSBoost infrastructure (INV-001).

### Star inflation — INV-001 link
Star accounts on affected repositories match bot heuristics from INV-001: zero activity, hash usernames, burst registration clusters. Same fake-engagement infrastructure used for social media fraud repurposed to make malware repositories appear trustworthy.

### Payload confirmed
GhostLoader: .NET dropper. RemcosRAT: full-featured RAT with keylogging, screen capture, file access, C2 communication.

### Scale
32+    repositories identified across network

2,100+ victim exposures confirmed

4      disclosure waves to GitHub Trust & Safety

4      delivery domains
